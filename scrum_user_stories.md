# 📄 Scrum User Stories
## 🌐 Astro Portfolio Website

---

## 🧑‍💻 Epic 1: Homepage & Navigation

### User Story 1.1 – View Homepage
**As a** visitor  
**I want to** see a visually appealing homepage  
**So that** I can understand the developer’s identity quickly  

**Acceptance Criteria:**
- Displays name, title, subtitle  
- Sky-blue background is visible  
- Page loads under 3 seconds  

---

### User Story 1.2 – Cloud Navigation
**As a** visitor  
**I want to** click cloud buttons  
**So that** I can navigate to different sections  

**Acceptance Criteria:**
- Cloud buttons visible (About, Skills, Experience, Projects, Contact)  
- Smooth scrolling works  
- Hover animation triggers  

---

### User Story 1.3 – Responsive Homepage
**As a** mobile user  
**I want to** view homepage properly  
**So that** I can navigate easily  

**Acceptance Criteria:**
- Layout adjusts for mobile/tablet  
- Cloud buttons stack vertically  
- Text remains readable  

---

## 💼 Epic 4: Experience Section

### User Story 4.1 – View Experience Timeline

**Acceptance Criteria:**

**Acceptance Criteria:**

---

## 📬 Epic 6: Contact Feature

### User Story 6.1 – Submit Contact Form
**As a** visitor  
**I want to** send a message  
**So that** I can contact the developer  

**Acceptance Criteria:**
- Form fields: name, email, message  
- Validation works  
- Success/failure message displayed  

---

### User Story 6.2 – Spam Protection
**As a** system  
**I want to** prevent spam  
**So that** the contact form remains secure  

**Acceptance Criteria:**
- CAPTCHA or validation applied  
- Invalid submissions blocked  

---

## ⚙️ Epic 7: Performance & Optimization

### User Story 7.1 – Fast Load Time
**As a** user  
**I want to** load pages quickly  
**So that** I have a smooth experience  

**Acceptance Criteria:**
- Load time < 3 seconds  
- Lighthouse score 90+  

---

### User Story 7.2 – Minimal JavaScript
**As a** system  
**I want to** load JS only when needed  
**So that** performance is optimized  

**Acceptance Criteria:**
- Astro Islands used  
- JS only for interactive parts  

---
